## Get Started in 5 Minutes

This guide will help you create and run your first Start UI [web] application quickly.

<Steps>
  <Step title="Create Your Project">
    Use the official CLI to scaffold a new project:

    ```bash
    pnpm create start-ui -t web myApp
    ```

    This creates a new directory `myApp` with the latest version of Start UI [web].

    <Note>
      Make sure you have Node.js 22 or later and pnpm installed on your system.
    </Note>
  </Step>

  <Step title="Navigate to Your Project">
    ```bash
    cd myApp
    ```
  </Step>

  <Step title="Configure Environment Variables">
    Copy the example environment file and update it:

    ```bash
    cp .env.example .env
    ```

    The default values work for local development. The most important variable to update is `AUTH_SECRET`:

    ```bash .env
    # Generate a secure secret for authentication
    AUTH_SECRET="your-secret-key-here"
    ```

    <Tip>
      Generate a secure secret using Better Auth CLI:
      ```bash
      npx @better-auth/cli@latest secret
      ```
    </Tip>

    <Accordion title="Key environment variables">
      - `VITE_PORT`: Development server port (default: 3000)
      - `DATABASE_URL`: PostgreSQL connection string
      - `AUTH_SECRET`: Secret key for authentication (required)
      - `GITHUB_CLIENT_ID` & `GITHUB_CLIENT_SECRET`: For GitHub OAuth (optional)
      - `DOCKER_DATABASE_PORT`: PostgreSQL port (default: 5432)
      - `DOCKER_MINIO_UI_PORT`: MinIO admin UI port (default: 9001)
      - `DOCKER_MAILDEV_UI_PORT`: Email testing UI port (default: 1080)
    </Accordion>
  </Step>

  <Step title="Install Dependencies">
    ```bash
    pnpm install
    ```

    This installs all required packages and runs post-install scripts to generate Prisma client and build info.
  </Step>

  <Step title="Start Docker Services">
    Start PostgreSQL, MinIO, and Maildev using Docker:

    ```bash
    pnpm dk:init
    ```

    This command starts Docker containers with the `--profile init` flag, which includes the initial bucket setup for MinIO.

    <Warning>
      **Don't have Docker?** You can use an external PostgreSQL database instead. Update the `DATABASE_URL` in your `.env` file and skip this step.
    </Warning>
  </Step>

  <Step title="Initialize Database">
    Push the Prisma schema and seed the database:

    ```bash
    pnpm db:init
    ```

    This runs two commands:
    - `pnpm db:push` - Creates database tables from your Prisma schema
    - `pnpm db:seed` - Populates the database with sample data

    <Accordion title="What gets seeded?">
      The seed script creates:
      - Demo admin user: `admin@start-ui.com` (password: `000000`)
      - Demo regular user: `user@start-ui.com` (password: `000000`)
      - Sample books with genres
      - Example data for testing features
    </Accordion>
  </Step>

  <Step title="Start Development Server">
    Run the development server:

    ```bash
    pnpm dev
    ```

    Your application will be available at:
    ```
    http://localhost:3000
    ```

    <Note>
      The `dev` command runs multiple processes in parallel:
      - Vite development server
      - Environment variable validation
      - Build info generation
    </Note>
  </Step>
</Steps>

## Explore Key Features

### OpenAPI Documentation

View auto-generated API documentation:
```
http://localhost:3000/api/openapi/app
```

This interactive interface lets you:
- Browse all backend endpoints
- Test API calls directly from the browser
- View request/response schemas

### Email Templates

Preview email templates in your browser:
```
http://localhost:3000/api/dev/email/login-code
```

You can also preview with different languages:
```
http://localhost:3000/api/dev/email/login-code?language=fr
```

### Database UI

Open Prisma Studio to view and edit database records:
```bash
pnpm db:ui
```

Access it at `http://localhost:5555`

## Common Commands

Here are the most frequently used commands:

```bash
# Development
pnpm dev              # Start dev server
pnpm db:ui            # Open Prisma Studio
pnpm storybook        # Start Storybook

# Docker
pnpm dk:start         # Start Docker containers
pnpm dk:stop          # Stop Docker containers
pnpm dk:clear         # Remove containers and volumes

# Database
pnpm db:push          # Push schema changes
pnpm db:seed          # Seed database

# Testing
pnpm test             # Run unit tests
pnpm test:ui          # Open Vitest UI
pnpm e2e:ui           # Open Playwright UI

# Code Quality
pnpm lint             # Run linters
pnpm pretty           # Format code
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port already in use">
    If port 3000 is already in use, change the `VITE_PORT` in your `.env` file:
    ```bash
    VITE_PORT=3001
    ```
    Also update `VITE_BASE_URL`:
    ```bash
    VITE_BASE_URL="http://localhost:3001"
    ```
  </Accordion>

  <Accordion title="Docker containers won't start">
    Make sure Docker is running and the ports aren't already in use:
    ```bash
    # Check if containers are running
    docker ps
    
    # Stop all containers and remove volumes
    pnpm dk:clear
    
    # Start fresh
    pnpm dk:init
    ```
  </Accordion>

  <Accordion title="Database connection error">
    Ensure PostgreSQL is running:
    ```bash
    # Check container status
    docker ps | grep postgres
    
    # View container logs
    docker compose logs postgres
    ```
    
    Verify your `DATABASE_URL` in `.env` matches your PostgreSQL configuration.
  </Accordion>

  <Accordion title="Prisma schema errors">
    If you encounter Prisma-related errors:
    ```bash
    # Regenerate Prisma client
    pnpm gen:prisma
    
    # Reset database (warning: deletes all data)
    pnpm db:push --force-reset
    pnpm db:seed
    ```
  </Accordion>
</AccordionGroup>
