Start UI [web] uses [React Email](https://react.email/) to build email templates with React components that render to HTML emails.

## Email Setup

### Email Server Configuration

Configure your email server in `.env`:

```bash .env
# For development (MailDev)
EMAIL_SERVER="smtp://0.0.0.0:1025"
EMAIL_FROM="Start UI <noreply@example.com>"

# For production (e.g., SendGrid)
EMAIL_SERVER="smtp://apikey:YOUR_SENDGRID_API_KEY@smtp.sendgrid.net:587"
EMAIL_FROM="Your App <noreply@yourapp.com>"
```

### Email Validation

Environment variables are validated in `src/env/server.ts`:

```typescript src/env/server.ts
export const envServer = createEnv({
  server: {
    EMAIL_SERVER: z.url(),
    EMAIL_FROM: z.string(),
  },
});
```

## Email Template Structure

### Email Layout Component

The base layout for all emails:

```tsx src/emails/components/email-layout.tsx
import { Body, Head, Html, Preview } from '@react-email/components';
import { ReactNode } from 'react';
import { AVAILABLE_LANGUAGES } from '@/lib/i18n/constants';
import { styles } from '@/emails/styles';

export const EmailLayout = ({
  preview,
  children,
  language,
}: {
  preview: string;
  children: ReactNode;
  language: string;
}) => {
  return (
    <Html
      lang={language}
      dir={
        AVAILABLE_LANGUAGES.find(({ key }) => key === language)?.dir ?? 'ltr'
      }
    >
      <Head>
        <meta name="viewport" content="width=device-width" />
      </Head>
      <Preview>{preview}</Preview>
      <Body style={styles.main}>{children}</Body>
    </Html>
  );
};
```

## Creating Email Templates

### Login Code Email

Real example from `src/emails/templates/login-code.tsx`:

```tsx src/emails/templates/login-code.tsx
import { Container, Heading, Section, Text } from '@react-email/components';
import i18n from '@/lib/i18n';
import { EmailFooter } from '@/emails/components/email-footer';
import { EmailLayout } from '@/emails/components/email-layout';
import { styles } from '@/emails/styles';
import { AUTH_EMAIL_OTP_EXPIRATION_IN_MINUTES } from '@/features/auth/config';

export const TemplateLoginCode = (props: {
  language: string;
  code: string;
}) => {
  i18n.changeLanguage(props.language);
  
  return (
    <EmailLayout
      preview={i18n.t('emails:loginCode.preview')}
      language={props.language}
    >
      <Container style={styles.container}>
        <Heading style={styles.h1}>
          {i18n.t('emails:loginCode.title')}
        </Heading>
        
        <Section style={styles.section}>
          <Text style={styles.text}>
            {i18n.t('emails:loginCode.intro')}
          </Text>
          
          <Text style={styles.code}>{props.code}</Text>
          
          <Text style={styles.textMuted}>
            {i18n.t('emails:loginCode.validityTime', {
              expiration: AUTH_EMAIL_OTP_EXPIRATION_IN_MINUTES,
            })}
            <br />
            {i18n.t('emails:loginCode.ignoreHelper')}
          </Text>
        </Section>
        
        <EmailFooter />
      </Container>
    </EmailLayout>
  );
};

export default TemplateLoginCode;
```

## Sending Emails

### Send Email Function

The email sending utility in `src/server/email.ts`:

```typescript src/server/email.ts
import { render } from '@react-email/render';
import nodemailer from 'nodemailer';
import { ReactElement } from 'react';
import { envServer } from '@/env/server';

const transporter = nodemailer.createTransport(envServer.EMAIL_SERVER);

export async function sendEmail({
  to,
  subject,
  template,
}: {
  to: string;
  subject: string;
  template: ReactElement;
}) {
  const html = await render(template);

  await transporter.sendMail({
    from: envServer.EMAIL_FROM,
    to,
    subject,
    html,
  });
}
```

### Sending from Better Auth

Integrate with Better Auth's email OTP:

```tsx src/server/auth.tsx
import { emailOTP } from 'better-auth/plugins';
import TemplateLoginCode from '@/emails/templates/login-code';
import { sendEmail } from '@/server/email';

export const auth = betterAuth({
  plugins: [
    emailOTP({
      async sendVerificationOTP({ email, otp, type }) {
        await match(type)
          .with('sign-in', async () => {
            await sendEmail({
              to: email,
              subject: i18n.t('emails:loginCode.subject', {
                lng: getUserLanguage(),
              }),
              template: (
                <TemplateLoginCode 
                  language={getUserLanguage()} 
                  code={otp} 
                />
              ),
            });
          })
          .exhaustive();
      },
    }),
  ],
});
```

## Internationalization

### Multi-language Support

Email templates support multiple languages:

```tsx
export const TemplateWelcome = (props: {
  language: string;
  userName: string;
}) => {
  i18n.changeLanguage(props.language);
  
  return (
    <EmailLayout
      preview={i18n.t('emails:welcome.preview')}
      language={props.language}
    >
      <Container>
        <Heading>
          {i18n.t('emails:welcome.title', { name: props.userName })}
        </Heading>
        <Text>
          {i18n.t('emails:welcome.body')}
        </Text>
      </Container>
    </EmailLayout>
  );
};
```

## Best Practices

<Steps>

<Step title="Use inline styles">

Email clients don't support external CSS. Always use inline styles:

```tsx
<Text style={{ fontSize: '16px', color: '#333' }}>
  Content
</Text>
```

</Step>

<Step title="Keep templates simple">

Avoid complex layouts. Email clients have limited CSS support:

```tsx
// Good: Simple layout
<Container>
  <Heading>Title</Heading>
  <Text>Content</Text>
</Container>

// Avoid: Complex flexbox layouts
<div style={{ display: 'flex', gap: '20px' }}>
  {/* Won't work in many email clients */}
</div>
```

</Step>

<Step title="Always include preview text">

Preview text appears in email clients before opening:

```tsx
<EmailLayout preview="Your login code is 123456" language="en">
  {/* Content */}
</EmailLayout>
```

</Step>

<Step title="Use absolute URLs for images">

```tsx
// Good
<Img src="https://example.com/logo.png" alt="Logo" />

// Bad
<Img src="/logo.png" alt="Logo" />
```

</Step>

<Step title="Test across email clients">

Test your emails in:
- Gmail
- Outlook
- Apple Mail
- Mobile clients

</Step>

</Steps>
