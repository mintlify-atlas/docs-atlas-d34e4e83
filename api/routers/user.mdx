The User router provides comprehensive endpoints for user management, including CRUD operations and session management.

## Authentication

All endpoints require authentication via `protectedProcedure` with specific permissions.

## Endpoints

### getAll

Retrieves a paginated list of users with optional search.

**Method:** `GET`

**Path:** `/users`

**Permissions:** `user:list`

#### Input Parameters

<ParamField path="cursor" type="string">
  Pagination cursor (user ID) for fetching the next page
</ParamField>

<ParamField path="limit" type="number" default="20">
  Number of items per page (min: 1, max: 100)
</ParamField>

<ParamField path="searchTerm" type="string" default="">
  Search term to filter users by name or email (case-insensitive)
</ParamField>

#### Response

<ResponseField name="items" type="User[]">
  Array of user objects
</ResponseField>

<ResponseField name="nextCursor" type="string">
  Cursor for the next page of results (if available)
</ResponseField>

<ResponseField name="total" type="number">
  Total number of users matching the search criteria
</ResponseField>

#### Example

```typescript
import { orpc } from '@/lib/orpc/client';

const result = await orpc.user.getAll({
  limit: 20,
  searchTerm: 'john',
});

console.log(`Found ${result.total} users`);
console.log(result.items);
```

### getById

Retrieves a single user by their ID.

**Method:** `GET`

**Path:** `/users/{id}`

**Permissions:** `user:list`

#### Input Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user
</ParamField>

#### Response

<ResponseField name="user" type="User">
  The user object with all fields
</ResponseField>

#### Example

```typescript
import { orpc } from '@/lib/orpc/client';

const user = await orpc.user.getById({
  id: 'user_123',
});
```

#### Errors

- `NOT_FOUND`: User with the specified ID does not exist

---

### create

Creates a new user.

**Method:** `POST`

**Path:** `/users`

**Permissions:** `user:create`

#### Input Parameters

<ParamField path="name" type="string">
  The user's name
</ParamField>

<ParamField path="email" type="string" required>
  The user's email address
</ParamField>

<ParamField path="role" type="string">
  The user's role (defaults to 'user')
</ParamField>

#### Response

<ResponseField name="user" type="User">
  The created user object
</ResponseField>

#### Example

```typescript
import { orpc } from '@/lib/orpc/client';

const newUser = await orpc.user.create({
  name: 'John Doe',
  email: 'john@example.com',
  role: 'user',
});
```

#### Behavior

- `emailVerified` is automatically set to `true`
- Default role is `user` if not specified

---

### updateById

Updates an existing user's information.

**Method:** `POST`

**Path:** `/users/{id}`

**Permissions:** `user:set-role`

#### Input Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user to update
</ParamField>

<ParamField path="name" type="string">
  The user's updated name
</ParamField>

<ParamField path="email" type="string" required>
  The user's updated email address
</ParamField>

<ParamField path="role" type="string">
  The user's updated role
</ParamField>

#### Response

<ResponseField name="user" type="User">
  The updated user object
</ResponseField>

#### Example

```typescript
import { orpc } from '@/lib/orpc/client';

const updatedUser = await orpc.user.updateById({
  id: 'user_123',
  name: 'Jane Smith',
  email: 'jane@example.com',
  role: 'admin',
});
```

#### Behavior

- Users cannot change their own role
- If email is changed, `emailVerified` is automatically set to `true`

#### Errors

- `NOT_FOUND`: User with the specified ID does not exist

---

### deleteById

Deletes a user by their ID.

**Method:** `DELETE`

**Path:** `/users/{id}`

**Permissions:** `user:delete`

#### Input Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user to delete
</ParamField>

#### Response

<ResponseField name="void" type="void">
  Returns void on success
</ResponseField>

#### Example

```typescript
import { orpc } from '@/lib/orpc/client';

await orpc.user.deleteById({
  id: 'user_123',
});
```

#### Behavior

- Users cannot delete themselves
- Removes the user from the authentication system

#### Errors

- `BAD_REQUEST`: User attempted to delete themselves
- `INTERNAL_SERVER_ERROR`: Failed to delete the user

### getUserSessions

Retrieves all sessions for a specific user.

**Method:** `GET`

**Path:** `/users/{userId}/sessions`

**Permissions:** `session:list`

#### Input Parameters

<ParamField path="userId" type="string" required>
  The unique identifier of the user
</ParamField>

<ParamField path="cursor" type="string">
  Pagination cursor (session ID) for fetching the next page
</ParamField>

<ParamField path="limit" type="number" default="20">
  Number of items per page (min: 1, max: 100)
</ParamField>

#### Response

<ResponseField name="items" type="Session[]">
  Array of session objects
</ResponseField>

<ResponseField name="nextCursor" type="string">
  Cursor for the next page of results (if available)
</ResponseField>

<ResponseField name="total" type="number">
  Total number of sessions for the user
</ResponseField>

#### Example

```typescript
import { orpc } from '@/lib/orpc/client';

const sessions = await orpc.user.getUserSessions({
  userId: 'user_123',
  limit: 10,
});
```

### revokeUserSessions

Revokes all sessions for a specific user.

**Method:** `POST`

**Path:** `/users/{id}/sessions/revoke`

**Permissions:** `session:revoke`

#### Input Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user
</ParamField>

#### Response

<ResponseField name="void" type="void">
  Returns void on success
</ResponseField>

#### Example

```typescript
import { orpc } from '@/lib/orpc/client';

await orpc.user.revokeUserSessions({
  id: 'user_123',
});
```

#### Behavior

- Users cannot revoke all their own sessions
- Logs out the user from all devices

#### Errors

- `BAD_REQUEST`: User attempted to revoke all their own sessions
- `INTERNAL_SERVER_ERROR`: Failed to revoke sessions

---

### revokeUserSession

Revokes a specific session for a user.

**Method:** `POST`

**Path:** `/users/{id}/sessions/{sessionToken}/revoke`

**Permissions:** `session:revoke`

#### Input Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user
</ParamField>

<ParamField path="sessionToken" type="string" required>
  The session token to revoke
</ParamField>

#### Response

<ResponseField name="void" type="void">
  Returns void on success
</ResponseField>

#### Example

```typescript
import { orpc } from '@/lib/orpc/client';

await orpc.user.revokeUserSession({
  id: 'user_123',
  sessionToken: 'session_abc',
});
```

#### Behavior

- Users cannot revoke their current session
- Logs out the user from a specific device

#### Errors

- `BAD_REQUEST`: User attempted to revoke their current session
- `INTERNAL_SERVER_ERROR`: Failed to revoke the session
