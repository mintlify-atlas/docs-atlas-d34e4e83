Start UI [web] includes a comprehensive set of UI components built on [@base-ui/react](https://base-ui.com) with custom styling and enhancements.

## Button Components

### Button

Primary button component with multiple variants and sizes.

```tsx
import { Button } from '@/components/ui/button';

<Button variant="default" size="default">
  Click me
</Button>

// With loading state
<Button loading={isLoading}>
  Submit
</Button>
```

**Variants**: `default`, `secondary`, `ghost`, `destructive`, `destructive-secondary`, `link`

**Sizes**: `xs`, `sm`, `default`, `lg`, `icon`, `icon-xs`, `icon-sm`, `icon-lg`

**Props**:
- `loading?: boolean` - Shows spinner and disables button
- `variant?: string` - Button style variant
- `size?: string` - Button size
- All Base UI Button props

### Button Link

Button styled as a link for navigation.

```tsx
import { ButtonLink } from '@/components/ui/button-link';

<ButtonLink to="/dashboard" variant="default">
  Go to Dashboard
</ButtonLink>
```

### Responsive Icon Button

Button that shows only icon on mobile, icon + label on desktop.

```tsx src/components/ui/responsive-icon-button.tsx
import { ResponsiveIconButton } from '@/components/ui/responsive-icon-button';
import { Plus } from 'lucide-react';

<ResponsiveIconButton label="Add Item" variant="default">
  <Plus />
</ResponsiveIconButton>
```

**Props**:
- `label: ReactNode` - Label text (required)
- `breakpoint?: number` - Custom mobile breakpoint
- `size?: 'xs' | 'sm' | 'default' | 'lg'`

## Form Input Components

### Input

Text input with optional start/end addons.

```tsx src/components/ui/input.tsx
import { Input } from '@/components/ui/input';
import { Search } from 'lucide-react';

<Input
  type="text"
  placeholder="Search..."
  startAddon={<Search />}
/>
```

**Props**:
- `startAddon?: ReactNode` - Icon/element before input
- `endAddon?: ReactNode` - Icon/element after input
- `size?: 'sm' | 'default' | 'lg'`

## Best Practices

<AccordionGroup>
  <Accordion title="Use semantic HTML">
    Components render appropriate semantic HTML elements. Avoid overriding with generic divs unless necessary.
  </Accordion>
  
  <Accordion title="Leverage variants">
    Use the built-in variant system rather than custom classes for consistency:
    
    ```tsx
    // Good
    <Button variant="destructive" />
    
    // Avoid
    <Button className="bg-red-500" />
    ```
  </Accordion>
  
  <Accordion title="Compose components">
    Build complex UIs by composing simple components:
    
    ```tsx
    <Card>
      <CardHeader>
        <CardTitle>Title</CardTitle>
      </CardHeader>
      <CardContent>
        Content
      </CardContent>
    </Card>
    ```
  </Accordion>
  
  <Accordion title="Use TypeScript">
    Take advantage of TypeScript for type safety and autocomplete.
  </Accordion>
</AccordionGroup>
