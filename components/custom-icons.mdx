Start UI [web] includes a custom icon generation system that converts SVG files into optimized React components using SVGR.

## Overview

The icon system provides:

- SVG to React component generation
- TypeScript support
- Automatic `currentColor` fill for theme compatibility
- Three variants per icon: regular, fill, and duotone
- Tree-shakable exports

<Info>
Icons are generated from SVG sources in `src/components/icons/svg-sources/` and output to `src/components/icons/generated/`
</Info>

## Icon Generation Workflow

### 1. Add SVG Source Files

Place your SVG files in the source directory:

```bash
src/components/icons/svg-sources/
├── icon-house.svg
├── icon-house-fill.svg
├── icon-house-duotone.svg
├── icon-user-circle.svg
├── icon-user-circle-fill.svg
└── icon-user-circle-duotone.svg
```

<Note>
Icon files must follow the naming convention: `icon-{name}.svg`, `icon-{name}-fill.svg`, `icon-{name}-duotone.svg`
</Note>

### 2. Generate React Components

Run the icon generation script:

```bash package.json:28
pnpm gen:icons
```

This command:
1. Reads all SVG files from `src/components/icons/svg-sources/`
2. Transforms them using SVGR with custom configuration
3. Outputs TypeScript React components to `src/components/icons/generated/`
4. Runs Prettier to format the generated code

### 3. Import and Use Icons

Generated icons are exported from the index file:

```tsx src/components/icons/generated/index.ts
export { default as IconHouse } from './icon-house'
export { default as IconHouseFill } from './icon-house-fill'
export { default as IconHouseDuotone } from './icon-house-duotone'
export { default as IconUserCircle } from './icon-user-circle'
export { default as IconUserCircleFill } from './icon-user-circle-fill'
export { default as IconUserCircleDuotone } from './icon-user-circle-duotone'
```

Use them in your components:

```tsx
import { IconHouse, IconHouseFill } from '@/components/icons/generated';

function Navigation() {
  return (
    <nav>
      <a href="/">
        <IconHouse className="size-5" />
        Home
      </a>
      <a href="/dashboard" aria-current="page">
        <IconHouseFill className="size-5" />
        Dashboard
      </a>
    </nav>
  );
}
```

## SVGR Configuration

The generation process uses a custom SVGR configuration:

```js src/components/icons/svgr.config.cjs
const comment =
  '// Autogenerated file, DO NOT MODIFY or it will be overwritten';

const template = (variables, { tpl }) => {
  const componentName = variables.componentName.replace('Svg', '');
  return tpl`
${comment}
${variables.imports};

${variables.interfaces};

const Svg = (${variables.props}) => (
  ${variables.jsx}
);

const ${componentName} = (props: SVGProps<SVGSVGElement>) => {
  return <Svg fill="currentColor" {...props} />;
};

export default ${componentName};
`;
};

module.exports = {
  icon: true,                              // Optimize for icon use
  outDir: './src/components/icons/generated',
  filenameCase: 'kebab',                   // kebab-case filenames
  expandProps: true,                       // Spread props
  typescript: true,                        // Generate TypeScript
  replaceAttrValues: { '#000': 'currentColor' },  // Theme support
  template,                                // Custom template
  prettier: false,                         // Run separately
};
```

### Key Configuration Options

- **`icon: true`** - Optimizes SVGs for icon usage (removes dimensions, adds viewBox)
- **`typescript: true`** - Generates TypeScript components with proper types
- **`replaceAttrValues`** - Replaces `#000` (black) with `currentColor` for theming
- **`template`** - Custom template that wraps SVG with `fill="currentColor"`

## Generated Component Structure

Each generated icon component looks like this:

```tsx src/components/icons/generated/icon-house.tsx
// Autogenerated file, DO NOT MODIFY or it will be overwritten
import type { SVGProps } from "react";
import * as React from "react";

const Svg = (props: SVGProps<SVGSVGElement>) => (
  <svg
    xmlns="http://www.w3.org/2000/svg"
    width="1em"
    height="1em"
    viewBox="0 0 256 256"
    {...props}
  >
    <path d="m219.31 108.68-80-80a16 16 0 0 0-22.62 0l-80 80A15.87..." />
  </svg>
);

const IconHouse = (props: SVGProps<SVGSVGElement>) => {
  return <Svg fill="currentColor" {...props} width="1em" height="1em" />;
};

export default IconHouse;
```

### Component Features

- **SVGProps** - Full TypeScript support for SVG attributes
- **fill="currentColor"** - Inherits text color from parent
- **width/height="1em"** - Scales with font size
- **viewBox** - Preserves aspect ratio
- **...props** - Forwards all props (className, style, etc.)

## Adding New Icons

### Step-by-Step Guide

1. **Find or create SVG icons**
   - Use icon libraries (Phosphor Icons, Heroicons, etc.)
   - Create custom SVGs in design tools
   - Ensure SVGs are optimized (use SVGO if needed)

2. **Prepare SVG files**
   - Clean up unnecessary attributes
   - Ensure paths use `#000` or `black` for fills (will be converted to `currentColor`)
   - Name files following convention: `icon-{name}.svg`

3. **Add to source directory**
   ```bash
   cp icon-star.svg src/components/icons/svg-sources/
   cp icon-star-fill.svg src/components/icons/svg-sources/
   cp icon-star-duotone.svg src/components/icons/svg-sources/
   ```

4. **Generate components**
   ```bash
   pnpm gen:icons
   ```

5. **Verify output**
   ```bash
   ls src/components/icons/generated/icon-star*
   # icon-star.tsx
   # icon-star-fill.tsx
   # icon-star-duotone.tsx
   ```

6. **Use in your app**
   ```tsx
   import { IconStar, IconStarFill } from '@/components/icons/generated';
   
   <IconStar className="size-5" />
   ```

### SVG Optimization Tips

<AccordionGroup>
  <Accordion title="Remove unnecessary attributes">
    Remove these if present:
    - `id` attributes
    - `class` attributes
    - `style` attributes
    - `data-*` attributes
    - Inline comments
  </Accordion>
  
  <Accordion title="Use relative units">
    Let SVGR handle sizing:
    - Remove fixed `width` and `height`
    - Keep `viewBox` attribute
    - Use `0 0 256 256` or similar for viewBox
  </Accordion>
  
  <Accordion title="Simplify paths">
    - Combine paths when possible
    - Remove invisible elements
    - Flatten transformations
  </Accordion>
  
  <Accordion title="Use currentColor">
    Icons work with theme colors:
    - Replace `fill="#000"` with `fill="currentColor"`
    - Or use `#000` (auto-converted by SVGR)
    - Remove hardcoded colors for flexibility
  </Accordion>
</AccordionGroup>


### When to Use Custom Icons vs Lucide

**Use custom icons for**:
- Brand-specific icons
- Icons not available in Lucide
- Icons requiring multiple variants (fill, duotone)
- Custom designs from your design team

**Use Lucide for**:
- Common UI icons (arrows, checkmarks, etc.)
- Quick prototyping
- Standard interface elements
- Large icon sets without custom requirements

## Best Practices

<AccordionGroup>
  <Accordion title="Consistent sizing">
    Use Tailwind's sizing utilities:
    
    ```tsx
    <IconHouse className="size-4" />  // 16px
    <IconHouse className="size-5" />  // 20px
    <IconHouse className="size-6" />  // 24px
    ```
  </Accordion>
  
  <Accordion title="Semantic naming">
    Name icons based on their meaning, not appearance:
    
    ```tsx
    // Good
    icon-home.svg
    icon-user.svg
    
    // Avoid
    icon-house-with-door.svg
    icon-person-silhouette.svg
    ```
  </Accordion>
  
  <Accordion title="Accessibility">
    Add labels for screen readers:
    
    ```tsx
    <button>
      <IconHome />
      <span className="sr-only">Go to home</span>
    </button>
    ```
  </Accordion>
  
  <Accordion title="Theme compatibility">
    Icons inherit color from parent:
    
    ```tsx
    <div className="text-primary">
      <IconHouse />  {/* Uses primary color */}
    </div>
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Icons not generating">
    Check:
    1. SVG files are in `src/components/icons/svg-sources/`
    2. Filenames follow `icon-*.svg` pattern
    3. Run `pnpm gen:icons` after adding files
    4. Check console for SVGR errors
  </Accordion>
  
  <Accordion title="Icons appear black instead of themed">
    Ensure SVG fills use `currentColor` or `#000`:
    
    ```xml
    <!-- Good -->
    <path fill="currentColor" d="..." />
    <path fill="#000" d="..." />
    
    <!-- Bad -->
    <path fill="#ff0000" d="..." />
    ```
  </Accordion>
  
  <Accordion title="TypeScript errors">
    Regenerate icons after updating SVGR config:
    
    ```bash
    pnpm gen:icons
    ```
  </Accordion>
  
  <Accordion title="Icons don't scale">
    Check that width/height are not hardcoded in SVG source:
    
    ```xml
    <!-- Good -->
    <svg viewBox="0 0 256 256">...</svg>
    
    <!-- Bad -->
    <svg width="256" height="256">...</svg>
    ```
  </Accordion>
</AccordionGroup>
